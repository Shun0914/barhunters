# 非機能要件（ドラフト）

数値は**デモ／MVP 向けの仮**。**本番前に SLO として再設定**すること。

---

## 1. 性能・拡張性

| ID | 項目 | 要件（案） | 備考 |
|----|------|------------|------|
| NFR-P-01 | 画面初期表示 | 主要画面が **3 秒以内**に First Meaningful Paint（デモ環境・想定データ量） | 計測方法: Lighthouse 等 TBD |
| NFR-P-02 | API 応答 | 一覧取得 **p95: 2 秒以内**（デモ） | 本番は負荷試験で更新 |
| NFR-P-03 | 同時利用者 | デモ: **数十ユーザー**想定 | 本番: TBD |
| NFR-P-04 | データ量 | 指標カード・申請が **年間数万件オーダー**まで設計余地 | 詳細はデータ要件 |

---

## 2. 可用性・信頼性

| ID | 項目 | 要件（案） | 備考 |
|----|------|------------|------|
| NFR-A-01 | デモ稼働 | 発表・レビュー時間帯の安定稼働 | 単一リージョン可 |
| NFR-A-02 | バックアップ | 本番では **RPO/RTO** を別途定義 | MVP 外でも可 |
| NFR-A-03 | エラー表示 | API 失敗時はユーザー向けメッセージを表示 | 技術スタックを露出しない |

---

## 3. セキュリティ

| ID | 項目 | 要件（案） | 備考 |
|----|------|------------|------|
| NFR-S-01 | 通信 | 本番は **HTTPS**（TLS 1.2+） | |
| NFR-S-02 | 認証情報 | パスワードを平文ログに出さない。保存時はハッシュ等 | 方式は実装時確定 |
| NFR-S-03 | セッション | 固定・推測困難なセッション ID、HttpOnly Cookie 等を検討 | |
| NFR-S-04 | 認可 | **ロールに応じた画面・API アクセス制御** | 経営閲覧のみ等 |
| NFR-S-05 | 入力検証 | XSS・SQL インジェクション対策（パラメータバインド、エスケープ） | |
| NFR-S-06 | 閉域 | 企画資料の **閉域 DB** 前提に合わせ、ネットワーク構成を設計 | |

---

## 4. 運用・保守

| ID | 項目 | 要件（案） | 備考 |
|----|------|------------|------|
| NFR-O-01 | ログ | 認証失敗・API エラー・主要操作の監査ログ（本番） | 個人情報を過剰に残さない |
| NFR-O-02 | デプロイ | ステージング／本番の環境分離（本番） | |
| NFR-O-03 | 設定 | 秘密情報は環境変数・Key Vault 等で管理 | |

---

## 5. ユーザビリティ・アクセシビリティ

| ID | 項目 | 要件（案） | 備考 |
|----|------|------------|------|
| NFR-U-01 | 言語 | UI **日本語**（Figma・企画資料に合わせる） | |
| NFR-U-02 | フォント | Noto Sans JP 等、可読性の高いフォント | Figma 準拠 |
| NFR-U-03 | アクセシビリティ | **WCAG 2.1 AA** を目標（本番）。デモは主要キーボード操作・コントラストを意識 | 段階的で可 |
