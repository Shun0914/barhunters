## 1. 方針

- **クライアント**は Web ブラウザ。主な利用画面は [01_基本要件定義書.md](./01_基本要件定義書.md) に準拠。
- **フロントエンド**と **バックエンド API** は分離し、API は JSON で連携する。
- **永続データ**はバックエンド経由でのみ読み書きする（フロントから DB に直接接続しない）。
- **インフラ**はクエスト推奨の **Azure** を前提に検討するが、**構成図・リソース名は未確定（TBD）**。

## 3. Azure 上での対応（案・要合意）

論理コンポーネントと **Azure サービスの対応イメージ**だけを置く。**SKU・リージョン・VNet / ピアリング等の詳細は本ファイルでは扱わない**（必要になったら別紙または将来 `docs/setup/` 側に分離する）。

| 論理コンポーネント | Azure での候補（例） | メモ |
|-------------------|----------------------|------|
| フロントエンド（Next.js） | Static Web Apps、App Service、Blob Storage + CDN など | ホスティング方式は未決定（TBD） |
| バックエンド API（FastAPI） | App Service、Azure Container Apps など | コンテナ化の要否で選択が変わりうる |
| データベース | Azure Database for MySQL（Flexible Server）、または Azure Database for PostgreSQL など | 論理スキーマは [04_データ要件.md](./04_データ要件.md)。製品選定は TBD |
| シークレット・接続文字列 | Azure Key Vault など | 本番では Key Vault 参照を推奨 |
| （任意）認証・トークン検証 | Microsoft Entra ID（Azure AD）連携、またはアプリ側セッションのみ など | MVP はシンプルで可 |
| （任意）エッジ・TLS 終端 | Azure Front Door、Application Gateway など | 単一 App Service だけで始める場合は省略もあり |
