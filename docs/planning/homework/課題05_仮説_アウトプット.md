## 仮説1

| 項目 | 内容 |
|------|------|
| **仮説本文** | もし「1P=10万円」を、**カテゴリ別の単価×件数**で積み上げるロジックにすれば、経営層は「その分の結果」を納得できる形で受け取れるはずである。 |
| **誰についての仮説か** | 経営層（経営会議で承認を取る判断をする人） |
| **この仮説を置く理由** | 中計で6,000P/10,000Pが既に掲げられている一方、1件あたりの定義が曖昧だと「証明」にならない。カテゴリ（1on1・資格・業務改善等）ごとに単価と件数を明示すれば、説明可能性が高まる。 |
| **この仮説が正しければ起こること** | 経営会議で「どういう効果がいくら分あるか」を説明でき、承認を得やすい。部門別・カテゴリ別の目標配分も議論しやすくなる。 |
| **この仮説が誤っていれば起こること** | 経営層が欲しているのは「合計P」や「トレンド」だけで、細かい積算は求めていない可能性がある。または、単価の根拠が弱く却下される。 |
| **何を確認できれば、仮説を前進／修正できるか** | 壁打ち①で「経営会議で見たい数字は何か（合計のみか、内訳・部門別・時系列か）」を聞く。必要なら単価の考え方（外部ベンチマーク・社内原価等）を一緒に整理する。 |

## 仮説2

| 項目 | 内容 |
|------|------|
| **仮説本文** | もし部門長の**入力負荷を「申請1件あたりの手間」で最小化**し、かつ経営には集計結果だけ見せる設計にすれば、現場の抵抗を抑えつつ効果の見える化は実現できるはずである。 |
| **誰についての仮説か** | 部門長（日々の運用・申請承認を行う人）と経営層（集計を見る人） |
| **この仮説を置く理由** | クエストで「部門長の負荷と現場の抵抗」が課題として挙がっている。入力が重いと形骸化し、軽くしすぎると効果の根拠が薄くなる。バランスの取り方が成否を分ける。 |
| **この仮説が正しければ起こること** | 部門長は負荷を許容し、申請が積み上がる。経営層はダッシュボードで効果を追え、両者が同じ制度を前提に動ける。 |
| **この仮説が誤っていれば起こること** | 負荷を下げるために項目を簡略化しすぎると、経営層が「証明」として見るに足る粒度にならない。または、部門長が「意味がない」と感じて申請を避ける。 |
| **何を確認できれば、仮説を前進／修正できるか** | 壁打ち①で部門長の業務実態（承認フローの現状・月あたりの件数感）を聞く。MVPでは「最低限どの項目を入力すれば経営用集計が成り立つか」を決め、負荷との両立を試す。 |

## 仮説3

| 項目 | 内容 |
|------|------|
| **仮説本文** | もし新しいポイント制度を**評価・人事査定と明示的に切り離し**（「評価には使わない」「見える化専用」と伝える）なら、公平性への不安と形骸化リスクは下がるはずである。 |
| **誰についての仮説か** | 現場・部門長（制度を「使う」側） |
| **この仮説を置く理由** | イシューツリーで「既存制度（MBO等）との整合」「公平性・形骸化防止」を重要論点にした。ポイントが評価に直結すると、付与の恣意性が疑われ、申請が歪む。 |
| **この仮説が正しければ起こること** | 制度が「効果の可視化」に特化し、評価は別軸と理解される。部門長も申請しやすく、経営層は効果データを説明責任に使える。 |
| **この仮説が誤っていれば起こること** | 切り離しても「いつか評価に使われるのでは」という不信が残る。または、評価と完全に切り離すと現場のインセンティブが弱く、申請が伸びない。 |
| **何を確認できれば、仮説を前進／修正できるか** | 人事制度・評価制度の現状（MBOの位置づけ、人事部の意向）をヒアリングする。制度設計時に「評価に使わない」を公式に打ち出すか、段階的に関連づけるかを検討する。 |

## 仮説4

| 項目 | 内容 |
|------|------|
| **仮説本文** | もし**部門別・カテゴリ別の目標配分**を経営層と事前に合意しておけば、6,000P/10,000Pは「上から押し付け」ではなく、部門が納得して動ける目標になるはずである。 |
| **誰についての仮説か** | 経営層（目標を設定する側）と部門（目標を実行する側） |
| **この仮説を置く理由** | 全社KPIだけでは部門が「自分ごと」にしづらい。クエストでも「部門別・カテゴリ別の目標配分」が効果試算の要素としてある。 |
| **この仮説が正しければ起こること** | 各部門が自部門の目標を認識し、申請・承認が計画的に進む。経営層は部門別の進捗で「どこが効いているか」を議論できる。 |
| **この仮説が誤っていれば起こること** | 配分の根拠が弱く部門から反発が出る。または、配分を決める前に制度設計を進める必要があり、順序の見直しが必要になる。 |
| **何を確認できれば、仮説を前進／修正できるか** | 壁打ち①で「部門別目標をどこまで経営がコミットするか」「まずは全社のみで始めるか」を確認する。QUEST2の効果試算で配分案を出し、経営の反応を見る。 |

## 仮説5

| 項目 | 内容 |
|------|------|
| **仮説本文** | もし**人的資本レポートや有報開示**と、今回の「効果の見える化」を**同じ指標・用語で接続**すれば、経営層は対外的な説明責任と社内の意思決定を一貫して語れるはずである。 |
| **誰についての仮説か** | 経営層・IR・人財戦略部（開示と経営判断の両方に関わる人） |
| **この仮説を置く理由** | 西部ガスHDは既に人的資本レポートを出している。内部の「証明」と外部への開示がバラバラだと、二重の説明負荷がかかり、経営の納得も得にくい。 |
| **この仮説が正しければ起こること** | 経営会議で使う数字と言葉が、有報・人的資本開示と一致し、説明が一本化する。人財戦略部の負荷も減る。 |
| **この仮説が誤っていれば起こること** | 開示用と内部管理用で指標が違うことが前提になっており、無理に揃えるとどちらも中途半端になる。 |
| **何を確認できれば、仮説を前進／修正できるか** | 現行の人的資本レポートで何を指標にしているか、有報で人的資本をどう記載しているかを確認する。壁打ち①で「今回の制度の数字を開示に使う想定か」を聞く。 |
