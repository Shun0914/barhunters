# 西部ガスHDクエスト文脈：下条案の数年実践ロードマップ

**これは何か**: [FigJam Sprint5（時間軸・各メンバー案）](https://www.figma.com/board/iKqXRvMqNU7EOZ3wAwIEg7/product-management?node-id=3182-1803&t=F1OfMQQTivzPW1Gh-4) と同じ発想で、**西部ガス社内に置いたときに「僕の案」をどう数年かけて実践するか**のたたき台。  
**僕の案の芯**（提案叩き台と同義）: **顧客価値 → 現場行動 → 日々の観測（部門長ダッシュボード）→ 経営・開示の言葉**を一本の鎖にする。長谷川さん案の「3価値×挑戦3サイズ」やポイント制度とは**接続するが代替しない**（ダッシュボード＝運用のハブ／物語の記録装置）。

**クエスト上の前提**（出典: `docs/research/Tech0_BtoC_Bootcamp_Step4_西部ガス様企業クエスト.md`）: 2026年5月経営会議で企画承認 → **2026年6月全社運用開始**。**2026年度 6,000P・2027年度 10,000P**（1P≒10万円相当の効果見える化）が経営コミットの置き場。ACT2027（〜2027年度）の人的資本KPIと**時間軸を噛み合わせる**。

---

## Phase 0｜クエスト期間〜起案直前（〜2026年5月）

| 観点 | 内容 |
|------|------|
| **やること（社内で）** | ① **顧客→行動→指標**の1枚図（仮説で可）と、**運用原則**（見える化と評価の距離）を、人財戦略部・先方と**言葉を揃える**。<br>② **部門長ダッシュボード**を、経営プレゼンに直結する形で **MVP以上**（クエスト仕様）まで持ち込む。<br>③ QUEST2 相当の **推進シナリオ＋効果試算**（6,000Pの「何をもって1Pか」の置き方を、ログ設計と矛盾なく接続）。 |
| **効果** | 経営会議資料が「制度案だけ」にならず、**6月から誰が何を週次で触るか**まで落ちる。**最終発表（5/20）と起案のストーリーが一致**する。 |
| **指標** | ・プレゼン内で **主要ユーザーフローがライブ**できる<br>・**先行指標の定義が3つ以上**文書化されている<br>・**「まだ決めていないこと」リスト**が共有されている（期待値コントロール） |
| **このフェーズでやらないこと** | 全部署の業務システム連携。**評価制度・労組合意**まで踏み込んだ確定。**実データに基づく因果の主張**（あくまで設計と試算）。 |

## Phase 1｜全社運用イヤー1（2026年6月〜2027年3月頃＝2026年度）

| 観点 | 内容 |
|------|------|
| **やること** | ① **顧客価値の一次分解**をワークで固定（部門ごとに言い換えてよいが、**社会貢献／安心安全／未来共創**など上位ラベルと突合）。<br>② ダッシュボードを **本番または準本番**で回し、**週次〜月次の振り返りの型**（部門長・課長）をオペレーションに埋め込む。<br>③ **パイロット→段階展開**の学びを、項目・文言・負荷に反映（形骸化の早期検知）。<br>④ **2026年度 6,000P** のストーリーに、**ログから説明できる切片**（件数・カテゴリ・部門アクティブ率など）を載せる。 |
| **効果** | 「給与を上げたが効果が見えない」に対し、**顧客に効く行動が増えている**という**社内説明の材料**が溜まる。現場は「人事の施策」ではなく**顧客のための記録**として読める余地ができる。 |
| **指標** | ・**部門長（＋承認者）アクティブ率**<br>・**週あたり記録件数**・**顧客価値タグ付きログの比率**<br>・**振り返り会の開催率**（定性でも可）<br>・既存の **エンゲージメント等**との**方向感**（因果は主張しない） |
| **このフェーズでやらないこと** | **人事評価への組み込み**。**先行・遅行の統計的証明**。**全社同一KPIでの部門ランキング公開**（歪みと抵抗を招きやすい）。 |

## Phase 2｜イヤー2・スケールと比較（2027年4月〜2028年3月頃＝2027年度）

| 観点 | 内容 |
|------|------|
| **やること** | ① **2027年度 10,000P** に向け、ログと試算モデルを**更新**（Phase1の実績を入力に）。<br>② **先行導入部門と未導入・遅れ部門の比較**（あれば）で、運用の型を改善。<br>③ **経営向け1枚**（四半期）を定例化：人的資本レポート・中計報告と**同じ用語**で並べられるようにする。<br>④ **とむさん案**（ストレッチゴール／公平性担保）など他施策がある場合、**同じ時間軸上で「いつ何がトリガーか」**を整理し、ダッシュボードは**記録・可視化の共通レイヤ**として接続。 |
| **効果** | 経営は「今年の投資で何が増えたか」を**年度単位で追える**。新中計への**組み込み候補**として扱える材料になる。 |
| **指標** | ・**YoY** のログ量・質（却下率・再掲率など運用指標があれば）<br>・**挑戦サイズ別・価値タグ別の内訳**（長谷川さんマトリクスと整合）<br>・**新規中計ドラフトに載せた指標の数**（プロセス指標） |
| **このフェーズでやらないこと** | **評価制度改定への直結**（条件未充足のまま押し込まない）。**心理的安全・生産性など二次KPIの主張**（測定と定義が未整備なら触れない）。 |

## Phase 3｜証明期・相関と開示準備（2028年4月〜2029年3月頃）

| 観点 | 内容 |
|------|------|
| **やること** | ① 蓄積ログと **経営指標・顧客指標**（JCSI 等、会社が既に追っているもの）の **相関・仮説検証**を開始（分析ユニットと役割分担）。<br>② **人的資本ROI** の社内定義を、西部ガス版に更新（レポート記載と整合）。<br>③ **有報・人的資本開示**に載せられる **定義済み指標** の候補を絞る。 |
| **効果** | 「見える化」が **説明責任のインフラ**になる。外部説明と内部運用の **二重手間**を減らす。 |
| **指標** | ・**分析サイクル**（四半期ごとの更新）が回っているか<br>・**開示候補指標**が **定義・限界**つきでリスト化されているか |
| **このフェーズでやらないこと** | **学会レベルの因果推論の完成**を待つこと。**評価への組込**は条件が揃うまで「検討」扱い。 |

## Phase 4｜制度接続・定着（2029年4月〜）

| 観点 | 内容 |
|------|------|
| **やること** | ① 相関・現場納得・基準の具体化など **前提条件**が揃った範囲で、**人事評価・Will Grow Fair** 等との接続を検討（Sprint5 の「結合する／しない」の枠を踏襲）。<br>② ダッシュボードを **業務システム・人事情報**と段階的に連携（コストとセキュリティの現実ラインで）。<br>③ **撤退・テコ入れ基準**（アクティブ率低下等）を運用に明文化。 |
| **効果** | 「やっていたら評価に反映された」ではなく **「現場が自分ごとで動いていたら、制度側が追いついた」**という納得の作り方が可能になる。 |
| **指標** | ・**接続した制度の数**より **現場の継続率・質指標**<br>・**労使合意プロセス**のマイルストーン達成 |
| **このフェーズでやらないこと** | **全指標の評価化**（ポイントやログのゲーム化）。**他社パッケージへのすり替え**でストーリーを失うこと。 |

## メンバー案との「横の置き方」（FigJam と揃える用）

| ボード上の塊 | 下条案との関係 |
|--------------|----------------|
| **長谷川さん**：ポイント・48時間・部門長アクセラ・失敗表彰・KPI/撤退 | **顧客価値タグ**と **挑戦3サイズ**を、ダッシュボードの **分類軸**として実装・運用で同期。ポイントの「中身」が顧客ストーリーで説明できるようにする。 |
| **とむさん**：ストレッチゴール／評価公平性 | **宣言→振り返り→承認**のデータがダッシュボードに流れ込む **将来接続**を Phase 2 以降で設計。Phase 0〜1 では **データ項目の互換**だけ意識してもよい。 |
| **クエストの 6,000P / 10,000P** | Phase 1〜2 の **経営説明の数値枠**。下条案は **「何をログに残すとPの説明に耐えるか」** の設計責務を持つ。 |
