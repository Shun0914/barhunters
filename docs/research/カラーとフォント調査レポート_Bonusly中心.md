# カラーとフォント調査レポート（Bonusly中心）

Bonusly と比較用として 15Five を併記。宿題の「HSBカラーパレット」「和文・欧文フォント」の整理や、自プロダクトのトークン設計の参考用。

**注意**: Bonusly のブランドは同社の知的財産です。本レポートは **公開マーケサイトのCSS**、**YouTubeデモ由来のキャプチャ**、および **無料トライアル等でログインした際の画面キャプチャの観察メモ** に基づくスタディであり、公式デザインキットの複製ではありません。スクショに個人名や組織情報が写る場合は、提出・共有時にマスキングを検討してください。

---

## 調査のしかた（Bonusly・再現手順）

1. **意図の文章**: [Bonusly, Enhanced: The Story Behind Our Transformed Brand](https://bonusly.com/post/brand-and-product-refresh) を読み、色・UI方針の説明を確認。デザイナー事例（例: Brad Matsushita のポートフォリオ）で「Highlighter」やコントラストの文脈を補足。
2. **実際の色・フォント（数値）**
  - `https://www.bonusly.com/` のHTMLを取得し、`link rel="stylesheet"` から **Webflow CDN の `bonusly.shared.*.min.css`** を特定。  
  - 同HTML内の `WebFont.load({ google: { families: ["Bricolage Grotesque:..."] }})` 等で **読み込みフォント** を確認。  
  - CSS全文から `:root{--gray-900:#...;...}` を抽出し、必要に応じて `#xxxxxx` の出現頻度を集計。  
  - HEX を **Python `colorsys.rgb_to_hsv`** などで **HSB** に換算。
3. **補足（実アプリを見る方法）**
   - **YouTube デモ**や公式紹介で、ログインに近いUIを確認できる。
   - **無料トライアルでログイン**すると、ナビ構成・AIアシスト・管理画面など、デモでは省略されがちな**実機のレイヤー**まで追える。
   - いずれも**スクショ**を残すと、宿題の「実際のトンマナ」とマーケの差分を議論しやすい。
4. **限界**: マーケCSSから拾ったHEXと**ピクセル一致**するかは、画質・圧縮の影響もある。**厳密なトークン**は、ログイン後の画面で**開発者ツール（Computed / スタイル）**を開くのが最も確実。

## 実アプリ（YouTubeデモ・スクショ）での観察

2026年4月時点、**YouTube上のデモ映像**から取得した画面キャプチャに基づくメモ（数値は目視・近似）。

**参考キャプチャ**: YouTube デモ・トライアル画面のスクショは、必要に応じて**チーム共有ストレージ等にローカル保管**する（本リポジトリには未同梱）。

### マーケサイトの調査結果との整合

- **主CTAが青・ロゴが緑**の役割分担は、スクショ上でも一致（マーケ記事の「主要操作は青」と同じ構造）。
- **ページ背景は明るいニュートラル、カードは白**で、マーケの `--base--gray` 系・ライトUI方針と矛盾しない。
- **紫**はハッシュタグ・アバターなど**アクセント**として使われており、`--base--purple` の用途イメージに近い。
- **フォント**: アプリ内は **Bricolage よりもニュートラルなサンセリフ**（Inter / Roboto 系に見える）が主体。**マーケ見出し用フォントとプロダクトUI用フォントが分かれている**可能性が高い（プロダクト側は読みやすさ優先のスタックと推測）。

### 画面ごとの印象（宿題・Figma用メモ）

| 観点 | ダッシュボード（`dashboard.png`） | Rewards（`rewards.png`） |
|------|-----------------------------------|---------------------------|
| レイアウト | カードグリッド＋上部ナビ | 左サイドバー（カテゴリ）＋右カードグリッド |
| 背景 | オフホワイト〜ライトグレー上に白カード | メインエリアは白基調。アクティブタブに淡い青紫のピル |
| コンポーネント | 「Give recognition」、@宛先・金額・#タグの**入力チップ**、丸アバター、+ptバッジ、細線アイコン | 報酬カード（角丸）、サイドバー検索、選択中は**黒背景＋白字**の強調行など |
| トーン | 余白多め・イラストや絵文字で**親しみ** | グリッドは整然。外部ブランド色とBonusly系ダークカードの**対比**で種類が分かる |

### YouTubeデモを使うときのコツ

- 検索ワード例: `Bonusly demo`, `Bonusly product tour`, `Bonusly tutorial`（公式チャネルや信頼できる解説動画を優先）。
- **一時停止して高解像度でキャプチャ**すると、HSB宿題用にスポイトで拾いやすい。

### 実アカウントでログインしたときの追加所見（2026-04）

デモ映像だけでは分かりにくい**現行プロダクトの構造**が確認できる。

| 観点 | `home_logged_in.png`（一般ホーム） | `admin_dashboard.png`（管理） |
|------|-------------------------------------|--------------------------------|
| レイアウト | **左サイドナビ**＋**中央のAIアシスト（Bizy）**＋**右メイン**の3カラムに近い分割 | **左：管理ナビ**（トライアル残日・会社設定など）＋**右：白カードのダッシュボード** |
| 色 | 白・ライトグレー基調、**青**がボタン・アイコン・アクティブの主軸。**緑**はポイントバッジ（例: Rewards の +10）など | **青**（アクティブ項目・マスコットアイコン）、**緑**（トライアル帯・Peer-to-peer カード）、**紫**（Celebrations カード）、**グレー**（ロック中機能） |
| コンポーネント | 「Who do you want to recognize?」の**太い入力バー**、フィードの **#ハッシュタグ**、**Say thanks / Like**、AIへの**提案ピル** | **参加率ゲージ**、**Program adoption** の4枚カード（達成／ロックの対比）、上部の**情報バナー** |
| トーン | **角丸を大きめ**にそろえた親しみやすいSaaS。フッター付近の**ハイタッチイラスト**で「堅すぎない」印象を補強 | 分析・設定寄りだが、カードとアイコンで**ゲーミフィケーション／福利厚生アプリ**の雰囲気が残る |

**マーケCSS・デモスクショとの関係**: ログイン後も **「青＝行動」「緑＝ポイント／前向きな状態」「紫＝特定モジュールのアクセント」** の整理は踏襲されているように見える。新しく目立つのは **AIチャット列**（プロダクトの主役が「称賛入力＋フィード」から「アシスト＋称賛」へ広がっている）こと。

**宿題・Figma向け**: スポイトで拾うなら、**サイドバー背景／選択中の青枠／トライアル帯の緑／白カードの影**をそれぞれ1色ずつメイン・アクセント・ニュートラルに割り当てると説明しやすい。

### 開発者ツールで分かったこと：プロダクトは Chakra UI のトークン

ログイン後の画面で `body` や `:root` を見ると、**[Chakra UI](https://chakra-ui.com/)** 由来の CSS 変数（`--chakra-colors-*` 等）が並ぶ。マーケの Webflow とは**別実装**だが、ニュートラルは近い値が出ている。

**ライトテーマのセマンティック色（抜粋）**

| 用途 | Chakra 変数の参照先 | 実体（貼り付けログより） |
|------|---------------------|---------------------------|
| 本文色 | `--chakra-colors-chakra-body-text` | `gray-800` → `#2d3039` |
| ページ背景 | `--chakra-colors-chakra-body-bg` | `white` |
| 枠線 | `--chakra-colors-chakra-border-color` | `gray-200` → `#c6cbd2` |
| 控えめな背景 | `--chakra-colors-chakra-subtle-bg` | `gray-100` → `#dbdfe6` |
| サブテキスト | `--chakra-colors-chakra-subtle-text` | `gray-600` → `#515661` |
| プレースホルダー | `--chakra-colors-chakra-placeholder-color` | `gray-500` → `#656c7b` |

**グレースケールの一端（`:root`）**

- `gray-25`: `#f6f6f9` → マーケサイトで拾った `--base--gray` / `#f6f6f9` と**同じ**（全体のキャンバス色の系統が揃っている）。
- `gray-50` … `gray-800` まで段階あり（ログでは `gray-800` まで表示、`Show all properties (526 more)` 以下に**ブランド色の青・緑・紫**なども含まれる想定）。

**フォント**

- `body` は `font-family: var(--chakra-fonts-body)`（別定義で `var(--chakra-fonts-ui)` も参照）。
- 貼り付けログだけでは**最終的なフォント名**は確定しない。DevTools の **Computed** で `font-family` の**解決後の値**を見るか、`:root` の `--chakra-fonts-body` の定義までスクロールすると特定しやすい。

**その他**

- フォーカスリング: `--chakra-ring-color: rgba(66, 153, 225, 0.6)`（Chakra 定番の**青系フォーカス**）。
- `html` に `system-ui, sans-serif` の記述もあり、フォールバックスタックの一部。

**レポートへの含意**: 宿題で「どう調べたか」と書くときは、**マーケ＝Webflow の独自変数**、**アプリ＝Chakra のセマンティックトークン**と**役割が違うがニュートラルは連続**、と書ける。

## 1. Bonusly（詳細）

### テキストや背景はどんな色だったか（明度・彩度の目安）

マーケサイトCSSの `:root` 変数（抜粋）を、**HSB** で要約した例。


| 役割      | 変数名（参考）                                          | HEX                                | HSBの印象                     |
| ------- | ------------------------------------------------ | ---------------------------------- | -------------------------- |
| 本文テキスト  | `--gray-900`                                     | `#1c1f21`                          | ほぼ無彩色に近い暗いグレー（S約15%、B約13%） |
| ページ背景   | `--base--gray`                                   | `#f6f6f9`                          | ごく低彩度のオフホワイト（S約1%、B約98%）   |
| ブランド緑   | `--green-100` / `--base--green`                  | `#118342`                          | 中〜高彩度の緑（S約87%、B約51%）       |
| 主アクション青 | `--blue-500` / `--base--blue`                    | `#1c6fee`                          | 高彩度・高明度の青（S約88%、B約93%）     |
| 紫アクセント  | `--base--purple`                                 | `#7b5ce0`                          | 中〜高彩度の紫（S約59%、B約88%）       |
| 淡い面     | 例 `--blue-50`, `--green-50`, `--green-alpha-100` | `#d8ecf8`, `#d9f2e3`, `#ecf6f1` など | 低〜中彩度のパステル                 |


公式説明の要点（要約）:

- 旧ロゴの**緑はコントラスト課題**があった。
- 刷新後はパレットを広げつつ**全体的にライト**にし、**ユーザー内容（称賛等）が背景から浮く**ようにした。
- **称賛・リワードなど主要操作は青**に寄せ、見つけやすさ・慣れ親しんだ操作感を狙う。

### フォント

- **見出し系**: Google Fonts **Bricolage Grotesque**（ウェイト 300–700 を読み込み）。
- **本文・UI寄り**: **Inter**。一部 **Plus Jakarta Sans** が使われている形跡（CSS上の表記は `Jakarta` など）。

### なぜこの色を選んでいるか、疑問に思ったところ

- **緑のブランド資産**と**CTAの青**の二系統は説明は通るが、**「ブランド色＝どれ？」**が初心者には分かりにくい可能性。
- **紫・水色・ミント**まで幅があり、**マーケの賑やかさ**と**プロダクトUIの静けさ**の切り分けは、デモスクショを見ると**プロダクト側は比較的ニュートラル**寄りにまとまっている印象（ただし画面によりリワードカードなど例外あり）。
- 「Highlighter」でユーザーコンテンツを照らす一方、**アクセントが増えるとフィードのカードと競合**しないかはコンポーネント設計次第。

### 参考にできそうなところ

- 背景を**極めて薄く**、本文を**真っ黒にしない暗灰**にする → 長文・ダッシュボードで目が疲れにくい方向。
- **ライトなキャンバス＋意味のあるアクセント**は、「見える化専用」の安心感の土台に使いやすい。
- **主要行動だけ高彩度の色に寄せる**のは、部門長の「記録する」「配分する」など**主導線1本**を決める参考。
- **見出しに性格・本文は読みやすいサンセリフ**の役割分離。

## 2. 15Five（比較・補足）

### テキストや背景はどんな色だったか

ブランドパートナー記事・一般的なブランド解説では、**ダークインディゴ系**と**バーミリオン／オレンジ赤**のコントラスト、**明るいニュートラル**で抑える構成がよく語られる。


| 例          | HEX       | HSBの目安               |
| ---------- | --------- | -------------------- |
| ダークインディゴ系  | `#1B0E3F` | 紫寄りの濃色（S約78%、B約25%）  |
| バーミリオン     | `#DB3801` | 高彩度の暖色（S約100%、B約86%） |
| オレンジ赤（別表記） | `#FF4B11` | 高彩度・高明度寄り            |


※プロダクト画面ごとに変動するため、**正確なUIトークンは該当画面のCSS確認**が必要。

### フォント（ブランド事例）

[Focus Lab の 15Five 事例](https://www.focuslab.agency/work/15five) では、見出しに **Cornbread Honey**、本文・UIに **Phantom Sans** という**二層**が紹介されている（カスタム／ライセンス字体寄りで、そのままMVPにコピーしにくい点あり）。

### なぜこの色か、疑問に思ったところ

- **強い暖色アクセント**は前進感が出る一方、**「評価・管理」の緊張感**も出しやすい。やらされ感を抑えたい命題とは**面積とトーン**の調整が必要そう。

### 参考にできそうなところ

- **濃色＝信頼・土台／暖色＝行動・ハイライト**の役割分担。
- **イラストとタイポで人間味**を足す方針は、「数字に出ない努力を拾う」トーン議論に接続しやすい。

## 3. 設問形式の対照表


| 項目                   | Bonusly                                        | 15Five                               |
| -------------------- | ---------------------------------------------- | ------------------------------------ |
| **テキスト・背景（明度・彩度など）** | 背景は超低彩度の明るいグレー、字は低彩度の暗灰。アクセントは高彩度の青・緑・紫を役割で分離。 | 暗いインディゴと高彩度のオレンジ赤の対比＋明るいニュートラル、が典型像。 |
| **疑問に思った点**          | ブランド緑とCTA青の二系統、アクセント種類の多さと「主色」の説明の仕方。          | 強い暖色がモチベーション寄りに見えすぎないか、文化差の調整。       |
| **参考にできそうな点**        | ライトUI、主操作の色の一本化、見出し／本文のフォント分離。                 | 信頼と行動の色の整理、タイポ＋イラストで柔らげる考え方。         |
