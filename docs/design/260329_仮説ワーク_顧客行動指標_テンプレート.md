# 仮説ワーク：顧客 → 行動 → 指標（柱 A〜E の中身を作る）

ばーはんたーずの FigJam「提案スキーマ」**柱 A** と **Phase 0 の1枚図**を、チームの言葉で具体化するためのシート。西部ガスHDクエストのストーリー（経営会議・6月全社）にそのまま貼れる形を目指す。

**補足**: 下の「発散例」は **議論用のたたき台**（ガス／インフラ事業の一般イメージで書いたもの）。**貴社の実態・用語に必ず差し替え**て使う。正解ではない。

---

## 「仮説」と「仮定」どっち？

| 言い方 | 意味 | 例 |
|--------|------|-----|
| **仮定** | 議論の**前提**。「こう置いて進める」＝外したら設計が変わる | 「部門長が月次で配分ポイントを握る」「評価とは切り離す」 |
| **仮説** | **検証したい主張**。「本当か？」をデータやパイロットで見る | 「褒め認めの件数が増えると、顧客対応の一次解決率が上がる」 |

1枚図では、**上から下は仮定→仮説の階段**にすると説明しやすい。  
（「仮説可」と書いてあるのは、**数字や因果はまだ確定でなくてよい**という意味。）

---

## 1枚図の骨格（4レイヤー）

「3層」どころか、**顧客価値 → 現場行動 → 先行 → 遅行**の **4つ**に分けると、下のコア表の列とそのまま対応する。

### 全体の流れ

```text
顧客価値  ──逆算──▶  現場行動  ──観測──▶  先行指標  ──ストーリーで接続──▶  遅行指標・経営指標
```

| レイヤー | 一言で |
|----------|--------|
| 顧客価値 | お客様が**何が嬉しいか**（シーンまで） |
| 現場行動 | そのために**誰が何をするか**（動詞で） |
| 先行指標 | **週・月で数える**もの（ログ・件数） |
| 遅行指標 | **事業・開示**で後から効いてくる指標（接続は仮説） |

---

### ① 顧客価値

**定義**  
お客様が**体感する良さ**。「安心」だけにせず、**いつ・誰が・何が嬉しいか**まで落とす。

**発散例**（採用必須ではない）

- 漏洩・異常時、連絡から現場到着・処置までの流れが説明できる  
- 見積・工事の説明が、専門用語なしで理解できる  
- 点検・工事後、現場がきちんと片づいている  
- 問い合わせのたびに担当が変わっても、説明がブレない  
- 新料金・制度の案内が、自分ごととして分かる  
- 供給停止・復旧の連絡が早い・一貫している  
- 顧客の設備変更に合わせた提案がタイムリー  

---

### ② 現場行動

**定義**  
上の価値を生む**日々の振る舞い**。「頑張った」ではなく**動詞＋対象**。

**発散例**（採用必須ではない）

- 部門を跨いで、引き継ぎメモを標準フォーマットで残す  
- 夜間・休日の一次対応の役割を明示し、カバーする  
- 顧客宅で説明した内容を短く記録し、次回担当に共有する  
- ヒヤリハットをチームで言語化する  
- マニュアルにないケースを FAQ 化して共有する  
- 工事・点検の前後チェックリストを、二名で確認する  
- コールセンターと現場で、用語・回答方針を週次ですり合わせる  

---

### ③ 先行指標

**定義**  
**週次・月次で数えられる**もの。ダッシュボードの「1ログ」や集計単位と揃える。

**発散例**（採用必須ではない）

- 「顧客価値タグ」付きの称賛・記録件数  
- 跨ぎ対応に関するログ件数  
- 説明・提案に関するログ件数  
- 参加した管理職・現場職の人数（ユニーク）  
- カテゴリ別の件数推移  
- 部門内での配分バランス（中央値・ばらつき）  
- 週次振り返りへの出席率  

※最初は**件数だけ**からでよい。

---

### ④ 遅行指標

**定義**  
売上・解約・クレーム・重大インシデント・NPS・一次解決率・開示用 KPI など。**因果は断定せず**、「いつ頃・どの経路で効きうるか」を一文で添える。

**発散例**（採用必須ではない）

- 供給・保安関連の苦情・ヒヤリの件数／再発  
- 顧客向け説明のトラブル系クレーム  
- 解約・休眠の動き（※他要因と混ざる旨を注記）  
- 人的資本開示や中計で追う数とのラベル対応  

---

## テーブル：行を足して埋める（コア）

**ルール**: 1行＝「顧客価値の切り口」1つ。まずは**3〜5行**でよい。

**下表の V1〜V5 は発散例**（そのまま採用せず、削る・統合・言い換えしてよい）。

| ID | 顧客価値（お客様にとっての良いこと） | それを支える現場行動（具体動詞） | 先行指標（何を数えるか） | 遅行指標への接続（仮説一文） |
|----|--------------------------------------|-----------------------------------|---------------------------|------------------------------|
| V1 | **異常・緊急時**に、連絡から現場対応までの流れが迷子にならない | 跨ぎで**担当・状況・次アクション**を短く引き継ぐ／夜間カバーを**事前に割り当てる** | 「緊急・跨ぎ対応」タグのログ件数／関与した職種数 | 重大インシデント・再発クレームの**削減ストーリー**（因果はパイロットで検証） |
| V2 | **説明が分かる**（料金・工事・安全・手続き） | 顧客向け説明を**テンプレ＋当日メモ**で次担当に渡す／難解な案件を**図解・一言要約**で残す | 「顧客説明・提案」タグのログ件数／共有されたメモ件数 | 説明不備系の問い合わせ・クレームの**減少仮説**（他因子ありと明記） |
| V3 | **約束が守られる**（日程・再訪・復旧見込み） | 工事・点検の**前後チェック**を二名確認／遅延時の**顧客連絡テンプレ**を使う | 「約束遵守・品質」タグのログ／チェック実施件数 | 再訪率・工期遅延関連の**顧客満足指標**との接続を検討 |
| V4 | **組織で握っている**感じがする（担当が変わってもブレない） | コールと現場で**週次15分の用語すり合わせ**／FAQを**週1で1件更新** | すり合わせ**実施回数**／FAQ更新件数／「横連携」ログ件数 | 一次解決率・振り込み問い合わせの**重複減**など（仮説） |
| V5 | **新しい制度・料金**を自分ごとで理解できる | 説明会資料を**現場向けに噛み砕く**／お客様の質問を**タイプ分類**して共有 | 制度・料金関連の**社内Q&A登録数**／説明支援ログ件数 | 誤解に基づく問い合わせ**ピークの緩和**（時系列で見る） |
| V6 | （チームで追加） | | | |

## 柱 A〜E とこのシートの対応（何を書けばいいか）

| 柱 | このシートで埋める場所 | 具体化のコツ |
|----|------------------------|--------------|
| **A 顧客価値の分解** | 上表の **顧客価値** 列＋社内ワークで言葉を統一 | 「抽象語1語」で終わらせず、**お客様のシーン**（いつ・誰が・何が嬉しい）を1行メモ。**発散の種**は上の箇条書きと V1〜V5 を参照 |
| **B 見える行動** | **現場行動** 列 | 「頑張る」禁止。**誰が・何をした**が第三者にも分かる動詞 |
| **C ダッシュボード** | **先行指標** 列＋「記録の単位」（1ログ＝何を意味するか） | 入力負荷と頻度をセットで決める（週1回／事象起きたら 等） |
| **D 経営の次の言葉** | **遅行指標への接続** 列＋「いつ効くか」 | 因果を断定しない。**中計・開示で使える語**に寄せる |
| **E 導入の現実性** | 表の外に **仮定リスト**（下記） | パイロット部門・巻き込む役職・禁止事項 |

---

## 仮定リスト（柱 E・設計の前提）

チェック式で埋める。外すとプロダクトが変わるものだけ。

- [ ] 評価・査定とは**明確に切る**（見える化専用）  
- [ ] 最初の記録主体は **部門長のみ**／**課長まで含む**／**リーダーまで**  
- [ ] パイロットは **◯部門・◯名規模**  
- [ ] システム連携は **しない**／**◯のみ**（最初）  
- [ ] 「1P＝10万円相当」の公式は **経営側の定義に従う**／**チームで仮置きする**

---

## 次のワークの進め方（60〜90分イメージ）

1. **顧客価値だけ**を付箋で出す（A）→ 類似統合して3〜5行にする  
2. 各行に **現場行動** を1つずつ付ける（B）  
3. **数え方**を決める（C）— 無理なら指標を小さくする  
4. 各行について **「それが増えたら経営に何の意味？」を1文**（D）  
5. **仮定リスト**で線引き（E）
