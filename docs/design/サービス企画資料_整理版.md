# サービス企画資料(整理版)

## 1. 役割構成と利用イメージ

### 企業の役割構成

経営層 → 部門長 → 課長 → 係長 → 一般社員

### 利用者の整理

| 立場 | ダッシュボード | 入力システム |
|------|----------------|--------------|
| 経営層 | 見る人 | — |
| 部門長 | 見る人 | 評価・承認する人 |
| 課長・係長 | 使う人 | 入力する人 / 評価・承認する人 |
| 一般社員 | 使う人 | 入力する人 |

### 利用イメージ(コミュニケーションの強度別)

1. **コミュニケーション小**: 社員が各自記入し、承認をもらう
2. **コミュニケーション中**: チームミーティングの1項目として取り入れる。部門長は月次で参加し、活動状況の確認とFBを行う
3. **コミュニケーション大**: 社員が各自記入し、事前申請。1階層上の人と1対1で会話しながら承認をもらう

### 重要ポイント:行動変容とコミュニケーション活性化のバランス

**仮説**

- **行動変容について**
  - 部門長への報告や評価はプレッシャーが強い
  - → 1階層上の上司への報告で、挑戦に関する会話へのハードルを下げる

- **コミュニケーション活性化について**
  - 1on1がベストだが、所属メンバー数によっては個別会話は負担が大きい
  - チームごとに一括で会話を行うことでチームの連携が強まる
  - → 体系や場面を考慮した設計にする必要がある

**気になるポイント**: 部門の役割構成、チームの人数、ミーティング形式、上司との関わり方

## 2. ペインポイント洗い出し

### 入力者(一般社員・中間層)のペインポイント

| ペインポイント | 解決の方向性 |
|----------------|--------------|
| 上司に自分がどのように評価されているか分からない | 記録が個人ページに蓄積され、承認状況が自分で確認できる |
| 失敗すると評価が下がる不安がある | 結果ではなく挑戦した回数・姿勢を表示する設計が安心感を生む |
| 自己アピールが苦手、どうすればよいか分からない | 記入フォーマットが「アピールの型」を提供し、苦手でも書ける |
| 「挑戦」の定義が分からず動けない | フォームの問いが挑戦の基準を示し、何を書けばよいか明確になる |
| 頑張りが人事評価に繋がる実感がない | 承認記録とダッシュボードが「見える証拠」となり、努力の蓄積が形になる |
| 異動・転職時に自分の実績を整理できない | 蓄積された挑戦記録が自分のポートフォリオになる |
| 他の人がどんな挑戦をしているか分からず水準感が掴めない | MTGでの共有やダッシュボードの事例一覧で同僚の活動が参考・刺激になる |
| 挑戦が孤立した行動になり続けにくい | チームで共有することで「自分だけじゃない」という感覚が生まれ継続しやすくなる |
| 自分が今どこを走っていて、チームや部の方向性と繋がっているか知りたい | 部門長の月次FBとダッシュボード上の方針ひも付きで文脈が補完される |
| 上司と話す機会がなく、気軽に相談できない | 記入という名目が定期接触を正当化し、業務外の会話が生まれやすくなる |
| 複数人だと遠慮が生まれ忌憚のない意見交換ができない | 1on1により遠慮のない対話を可能にする |
| 挑戦の意味付けができず、なぜやるのか言語化できない | 対話の中で上司が引き出すことで自己理解が深まる |
| 次の一手が見えない、記録して終わりになる | 1on1でその場で次のアクションを一緒に設定し記録に残る |
| 自分の成長が実感できない | 1on1で言語化した内容がダッシュボードに蓄積され振り返りの材料になる |

### 評価・承認者(中間層・部門長)のペインポイント

| ペインポイント | 解決の方向性 |
|----------------|--------------|
| 安定した評価をしたい(定例で比較・蓄積したい) | 定例記録の蓄積で時系列比較がしやすくなる |
| 評価の説明責任を果たしにくい | 承認記録・活動ログが評価の根拠として機能する |
| 承認を後回しにしてしまう | 通知・リマインド機能で承認漏れを防ぎ、入力者の動機を守る |
| 上位層への報告資料の作成が手間 | ダッシュボードのデータをそのまま報告に活用できる |
| チーム全体の活性度が把握できない | ダッシュボードが俯瞰手段となり、誰が動いて誰が止まっているかが一覧できる |
| 業務外でどんなことをしているか・何に興味があるか知りたい | 記入内容の「挑戦の背景・動機」がダッシュボードで蓄積・参照できる |
| 部下の挑戦が部の方針と連動しているか確認できない | MTGのFB内容とダッシュボードの方針タグを組み合わせると整合確認が効率化される |
| 部下と接する機会が少なく活動を把握しきれない | MTGを議題化することで全員の活動状況を構造的に把握できる |
| 部下の文脈を知らずに指示・評価してしまう | 挑戦の背景を事前に把握することで、指示の質と部下の納得感が上がる |
| どんな指示が最適か、何を褒めればよいか分からない | 文脈を踏まえた承認になるため、表面的でないFBが自然にできる |
| 各個人の特性を理解できていない | 定期的な対話で業務外の関心や価値観が自然に見えてくる |
| 優秀な部下を引き留める材料が乏しい | 対話と記録の蓄積が「この会社で成長できる」という実感を作り定着につながる |

## 3. ペルソナ詳細(6名)

### ① 一般社員 / エネルギー統括部
- **性格**: 慎重だけど好奇心旺盛、裏でコツコツやる、圧力が苦手、自由にやらせると伸びる
- **習慣**: SNSをよく見るインドア派
- **特性**: 保守的・安定思考、自分のテリトリーを大事にする、線引きが上手
- **課題**: 自分に自信を持てない、考えすぎる、説明が下手

### ② 一般社員 / 事業開発部
- **性格**: 新しいことに関心がある、効率重視、先頭に立って進める、疑問から考えを深める
- **習慣**: ソロ活アウトドア派
- **特性**: 自分のルールに沿って生活する
- **課題**: 事前の根回しが難しい、視野が狭くなりがち、柔軟な対応ができない

### ③ 中間層 / 総務部
- **性格**: 納得するまで詰める、面倒見が良い、興味がはっきりしている、自分に自信がある
- **習慣**: ゲーム好きなインドア派
- **特性**: 大枠をととのえて指示を出す、センスを重視、綺麗好き
- **課題**: 人に合わせるとペースが乱れる、完璧主義、聞くだけでは理解できない

### ④ 中間層 / デザインマーケティング部
- **性格**: 楽観的、コミュニケーションが好き、不要と思ったらやらない、自分に自信がある
- **習慣**: コンテンツ大好きアウトドア派
- **特性**: 面白いこと優先、発想力を重視
- **課題**: 細かいところまでチェックできない、人に厳しく言えない、こだわりが強い

### ⑤ 部門長 / 不動産統括部
- **性格**: 現場重視、一歩引いて物事を見る、関心がはっきりしている、フットワークが軽い
- **習慣**: おでかけ(お酒)好きアウトドア派
- **特性**: 人が好き、良い関係性を築くのが早い
- **課題**: 強く意見を伝えられない、経営層と現場の板挟み、オーバーワーク、抽象的な課題を具体作業に落とせない

### ⑥ 部門長 / 経営戦略部
- **性格**: 積極的、面倒見が良い、学ぶことが好き、諦めがはやい
- **習慣**: 飲み会大好きアウトドア派
- **特性**: 自然とみんなの中心にいる
- **課題**: 予定を詰め込みすぎて時間がない、一目でわからないとそのままにする、説明が抽象的

## 6. Lean Canvas(ペルソナ&利用イメージ部分)

### ダッシュボード
- **使う人**: 部門長(課長・係長・一般社員)
- **見る人**: 経営層

### 入力システム
- **入力する人**: 課長・係長(部門長)
- **評価・承認する人**: 負担を分散させるのであれば1階層上の社員

### 利用イメージ
- 係長と一般社員が1on1で会話しながら記入
  - 1on1は利用イメージの一つであるとともに、ダッシュボードと同様、売上に寄与する指標としても機能する
  - 上司に意見を聞いてもらう場としての安心感、上司にアピールするための情報整理や特徴探索の場
- チームミーティングの1項目として取り入れる
- 社員が各自記入し、(1on1しながら)承認をもらう
- チームごと定例ミーティングに部門長が参加し、会話する

### 行動変容とコミュニケーション活性化のバランス
- **行動変容**: 部門長への報告・評価はプレッシャーが強すぎないか。1階層上の上司のほうが挑戦に関する会話のハードルが低い
- **コミュニケーション活性化**: 1on1がベスト。メンバー数によっては個別会話の負担が大きいため、チームごとに一括で会話し活動状況にFBする方法も検討

### 施策スコープ:対象とする社員のパターン
- やる気はあるけど時間・機会がない層
- 指示されたことはできるけど、挑戦が怖い層
