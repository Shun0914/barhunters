## 実装ルール

- **`dashboard/` はモック・静的プロトタイプ専用** — 本番実装の編集先にしない。新規開発は `frontend/` / `backend/` で行う。
- **技術スタック・アーキテクチャの正は [docs/requirements/05_アーキテクチャ.md](docs/requirements/05_アーキテクチャ.md)** に置く。CLAUDE.md には複製せず、必要な都度そちらを参照すること（情報の二重管理を避けるため）。
- **フロントエンドから DB へ直接接続しない**。永続データはバックエンド API 経由でのみ読み書きする。

---

## バックエンドの実行環境

バックエンド（FastAPI 想定）の実行には、**Anaconda の仮想環境 `barhunters` を使用する**。

- ライブラリのインストール（`pip install` / `conda install` 等）は **必ず `barhunters` 環境内で実行する**こと。
- バックエンドの起動（`uvicorn` / `python -m ...` 等）も **必ず `barhunters` 環境内で実行する**こと。
- グローバル環境や別の仮想環境にインストールしないこと。

### 環境のアクティベート

```bash
conda activate barhunters
```

`Bash` ツールで backend 関連のコマンドを実行する際は、`conda activate barhunters && <command>` の形で連結するか、事前にアクティベート済みであることを確認すること。

## Figma の扱い

- UI の正は Figma ファイル `nEqeLUmA2uW8vWINJNO01U`。デザインノード ID と画面の対応は [docs/requirements/01_基本要件定義書.md](docs/requirements/01_基本要件定義書.md) §5 を参照。
- Figma URL を渡された場合、または Figma 上のデザインに基づく実装が必要な場合は、MCP ツール `mcp__claude_ai_Figma__get_design_context` を優先的に利用する。
- 出力されるコードはあくまで **参照** であり、本リポジトリの規約・既存コンポーネントに合わせて適応すること。
- トンマナと比率はFigmaのデータを忠実に再現すること。例えばポイント申請フォームだとサイドバーとヘッダーの背景は#faf8f5、メイン画面の背景は#fff。その中にフォームがあり、背景色が#ecf5faになっている。
